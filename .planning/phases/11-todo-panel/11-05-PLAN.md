<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: 端到端手工 UAT 验证</name>
  <what-built>
    Phase 11 全量功能（11-01~11-04）：
    - 后端：GET /api/sessions/{id}/messages 响应加 todos 字段
    - 前端：Todo 抽屉面板（drawer + list + item + toggle-button）、ui-store（persist）、chat-store.todos、SSE todo listener、chat header toggle 按钮、AppLayout 条件式 grid
  </what-built>
  <how-to-verify>
    **前置准备：**
    1. 启动后端：`cd docker && docker-compose up -d` → `cd backend && python -m app.main`
    2. 启动前端：`cd frontend && npm run dev`
    3. 浏览器打开 `http://localhost:3000`，登录/进入默认用户上下文
    4. 准备两个浏览器标签或 DevTools Performance 工具（用于慢速动画播放）

    ---

    **验证点 A：首次 auto-open 行为**（CONTEXT.md D-02 硬约束）
    1. 新建会话
    2. 输入"帮我规划明天的一天行程，分三步"之类触发 Agent `write_todos` 的指令
    3. 观察：**drawer 应当首次 todo 事件到达时自动弹出一次**（grid 平滑过渡到 3 列；右侧出现任务计划面板）
    4. 期望结果：面板弹出，显示 todos 列表；toggle 按钮 `aria-pressed="true"`

    ---

    **验证点 B：手动关闭后不再自动弹**（D-02 红线）
    1. 在验证点 A 的同一会话内，点击 drawer header 的 × 按钮手动关闭
    2. 继续对话，引导 Agent 再次调用 `write_todos`（例如请求"调整一下这些任务"）
    3. 观察：**drawer 保持关闭**（grid 保持 2 列，面板不再自动弹出）
    4. 期望结果：toggle 按钮 `aria-pressed="false"`，手动点击 toggle 按钮才会打开

    ---

    **验证点 C：切换会话 todos 跟随（不自动弹）**（D-06 + A1 硬约束）
    1. 保持 drawer 关闭（从验证点 B 结束状态继承）
    2. 侧边栏切换到**有历史 todos 的另一会话**（若没有，先在两个会话分别触发过 write_todos 再来做本验证）
    3. 观察：
       - Drawer **不自动弹出**（A1 语义：loadHistory 路径不触发 auto-open）
       - 手动点 toggle 按钮打开后，面板显示**目标会话的 todos**（不是残留的旧会话 todos）
    4. 切换回另一会话 → 同理，toggle 打开后显示该会话 todos

    ---

    **验证点 D：删除当前会话跟随下一条**（D-07 + Phase 10 D-12 耦合）
    1. 有多个会话，当前活跃会话的 drawer 打开且有 todos
    2. 侧边栏删除当前会话
    3. 观察：面板跟随切换到下一会话的 todos 快照（不保留旧的、也不清空成空态）
    4. 反之：删除后若落到空态（没有其他会话），面板清空为空态文案"Agent 尚未制定任务计划"

    ---

    **验证点 E：reattach 幂等**（D-14 + RESEARCH.md Pitfall）
    1. 在 Agent `write_todos` 刚完成、todos 已渲染的会话上，强制断网（DevTools Network → Offline）
    2. 断 5 秒后恢复在线
    3. 观察：SSE reattach（`from_id=0`）重放历史事件，包括 todo 事件
    4. 期望结果：**面板内容不重复、不错乱**，最终 todos 仍为最新快照（setTodos 幂等覆盖）

    ---

    **验证点 F：DESIGN.md token 视觉走查**（D-08 + DESIGN.md 硬约束）

    打开 Chrome DevTools → Elements → 选中 TodoItem 内三态图标，检查 Computed 样式：

    - **pending 空心圆**：`border: 1.5px solid rgba(255, 255, 255, 0.08)`（即 `var(--color-border-standard)` 展开值）
    - **in_progress 半圈 spinner**：SVG `path` 的 `stroke: rgb(94, 106, 210)` 或 `stroke: #5e6ad2`（即 `var(--color-accent)`）；外层 `<svg>` 持续 `transform: rotate(...)` 动画
    - **completed 实心圆**：`background-color: rgb(94, 106, 210)`（`var(--color-accent)`）；内部 Lucide `<Check>` `color: rgb(255, 255, 255)` 或 `color: white`；尺寸 12×12（`size={12}`）
    - **文案颜色不变**：切换三态前后，`<span>` 的 `color: rgb(208, 214, 224)`（`var(--color-text-secondary)`）保持不变——即使 completed

    ---

    **验证点 G：动效时长走查**
    1. DevTools → Performance → Record → 在对话中触发一次 write_todos
    2. 或者 DevTools → Rendering → Enable "Paint flashing" + 降低动画速度
    3. 验证：
       - **新 todo item 进入**：opacity 0→1 + translateY 4→0，~200ms ease-out（`animate-[todoEnter_200ms_ease-out]`）
       - **状态切换图标**：pending→in_progress 或 in_progress→completed 时，~150ms CSS transition（`transition-all duration-150`）
       - **抽屉 toggle**：点 toggle 按钮时 grid 过渡 ~200ms ease-out（`transition-[grid-template-columns] duration-200 ease-out`）
       - **in_progress spinner**：持续旋转，`animationDuration: "1s"` linear

    ---

    **验证点 H：单 in_progress 约束**（D-08 + deepagents 语义）
    1. 在若干轮对话中保持面板打开
    2. 每次 Agent 更新 todos 时，截屏面板
    3. 确认：**任意时刻至多一条 item 是 in_progress**（deepagents 保证，前端不校验——若发现多条并发，记录 bug 到 11-05-SUMMARY.md，但**不归为 Phase 11 scope**，属 backend/agent 层面）

    ---

    **验证点 I：全量回归冒烟**
    1. `cd backend && pytest` → 全绿
    2. `cd frontend && npm run test -- run && npm run build` → 全绿
    3. 浏览器控制台：没有 React hydration 警告（skipHydration + rehydrate 正确实施的证据）
    4. 浏览器刷新页面：drawer 保持用户最后一次的 open/close 状态（persist 证据）；但 hasAutoOpenedFor 重置（刷新后若 Agent 首次 write_todos 会重新 auto-open）
  </how-to-verify>
