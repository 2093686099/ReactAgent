更新 Sparkles 完成图标逻辑：当消息中有 pending hitl segment 时不显示 Sparkles（消息未完成，正在等待审批）。在渲染 Sparkles 处加条件：
```typescript
const hasPendingHitl = message.segments.some(
  (s) => s.type === "hitl" && s.status === "pending"
);
// ...
{!isStreaming && !hasPendingHitl ? (
  <div className="mt-2 text-[var(--color-text-quaternary)]">
```

更新 memo 比较函数：当 message 中包含 `hitl` segment 且其 status 为 `pending` 时，不跳过 rerender（因为回调 props 可能变化）。简单做法：在已有 memo 逻辑的基础上，检查 segments 中是否有 pending hitl segment：
```typescript
export const MessageBubble = memo(
  MessageBubbleInner,
  (prev, next) => {
    if (next.isStreaming) return false;
    // HITL pending 状态需要响应回调变化
    const hasPendingHitl = next.message.segments.some(
      (s) => s.type === "hitl" && s.status === "pending"
    );
    if (hasPendingHitl) return false;
    return (
      prev.message.id === next.message.id &&
      prev.message.segments.length === next.message.segments.length
    );
  }
);
```
