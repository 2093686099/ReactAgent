## Typography

Phase 08 uses 5 roles from the DESIGN.md hierarchy. No display/heading sizes needed (no marketing content).

| Role | Size | Weight | Line Height | Letter Spacing | Font | Usage |
|------|------|--------|-------------|----------------|------|-------|
| Body | 15px | 400 | 1.60 | -0.165px | Inter Variable | AI response text, user message text |
| Body Medium | 15px | 510 | 1.60 | -0.165px | Inter Variable | Sidebar "新建会话" button label, input placeholder |
| Caption | 13px | 510 | 1.50 | -0.13px | Inter Variable | Timestamps, streaming status indicator text |
| Label | 12px | 510 | 1.40 | normal | Inter Variable | Error toast text, logo mark label |
| Mono Body | 14px | 400 | 1.50 | normal | Berkeley Mono | Code blocks in Markdown rendering |

Font feature settings on ALL Inter text: `font-feature-settings: "cv01", "ss03"`

**Rationale:** Chat UI is a reading interface. 15px body (DESIGN.md "Small" tier) is the right density for conversational text — 16px feels too large in a chat column, 14px too small for sustained reading. Weight 510 for UI labels per Linear's signature system.

**Source:** DESIGN.md Section 3 (Typography Rules)

## Interaction Contracts

### CHAT-01: Message Send

| State | Visual | Interaction |
|-------|--------|-------------|
| Idle | Input field with placeholder "输入消息..." | Textarea focused, cursor blinking |
| Composing | Text in input, send button enabled (`#5e6ad2` bg) | Enter to send, Shift+Enter for newline |
| Empty input | Send button disabled (`#62666d` icon, no bg) | Enter does nothing |
| Sending | User message appears immediately in list, input clears, send button disabled | POST /invoke fires, status -> streaming |

### CHAT-02: Streaming Response

| State | Visual | Interaction |
|-------|--------|-------------|
| Waiting | StreamingDots (3 pulsing dots) below user message | Non-interactive, auto-scroll follows |
| Streaming | AI text appears character-by-character (RAF batched) | Auto-scroll to bottom, user can scroll up to disengage |
| Complete | Full AI response with Markdown rendered, AI logo at bottom | Scroll position maintained |

**Auto-scroll behavior:**
- Auto-scroll is ON by default when streaming starts
- If user scrolls up (more than 100px from bottom), auto-scroll disengages
- When user scrolls back to bottom, auto-scroll re-engages
- Use `scrollTop = scrollHeight` inside RAF, not `scrollIntoView`

### CHAT-03: Markdown Rendering

| Content | Rendering | Notes |
|---------|-----------|-------|
| Paragraphs | 15px body text, `#d0d6e0` | Standard paragraph spacing (margin-bottom 8px) |
| Code blocks | Berkeley Mono 14px, `#191a1b` bg, `rgba(255,255,255,0.08)` border, 6px radius | rehype-highlight for syntax coloring |
| Inline code | Berkeley Mono 14px, `rgba(255,255,255,0.05)` bg, 2px radius, 2px 6px padding | Inline within text flow |
| Links | `#7170ff` text, underline on hover | Open in new tab (`target="_blank"`) |
| Lists | 15px body, 8px gap between items, 16px left indent | Ordered and unordered |
| Tables | `rgba(255,255,255,0.05)` header bg, `rgba(255,255,255,0.08)` cell borders | GFM tables via remark-gfm |
| Headings (in AI) | 15px weight 590 (same size, bolder — no size jump in chat) | AI response headings are emphasis, not layout |

**Streaming vs Complete rendering:**
- During streaming: render raw text with minimal formatting (bold, italic, links only)
- After `done` event: full Markdown pass with code highlighting, tables, complete blocks
- Throttle: if implementing mid-stream Markdown, limit to every 300ms max

### CHAT-06: Loading State

| State | Visual |
|-------|--------|
| Sending (POST in flight) | Send button shows spinner (lucide `Loader2` icon, animate-spin) |
| Waiting for first token | StreamingDots component: 3 dots with staggered opacity pulse animation |
| Streaming | StreamingDots removed, AI text accumulating |

StreamingDots CSS animation:
```
@keyframes dot-pulse {
  0%, 80%, 100% { opacity: 0.3; }
  40% { opacity: 1; }
}
dot-1: animation-delay 0ms
dot-2: animation-delay 160ms
dot-3: animation-delay 320ms
```
Dot size: 6px circle, color `#8a8f98`

### CHAT-07: Error State

| Error | Visual | Recovery |
|-------|--------|----------|
| POST /invoke fails (network) | Sonner toast: "发送失败，请检查网络连接" | User message remains in input (not cleared) |
| POST /invoke fails (server 5xx) | Sonner toast: "服务暂时不可用，请稍后重试" | User message remains in input |
| SSE error event | Inline error below AI message: "Agent 执行出错：{message}" in `#ef4444` text | No auto-retry; user can send new message |
| SSE connection drop | Silent — no visible error during Phase 08 (reconnection deferred to Phase 12) | Status returns to idle |

### Overall Layout (AppLayout)

```
+------------------+------------------------------------------+
|                  |                                          |
|  Sidebar         |  Chat Area                               |
|  240px fixed     |  flex-1                                  |

|                  |                                          |
|  [Logo]          |  +------------------------------------+  |
|                  |  | Message List (scroll)              |  |
|  [新建会话]       |  | max-width: 768px, centered         |  |
|                  |  | padding: 0 24px                    |  |
|                  |  |                                    |  |
|                  |  |  [User msg]          right-aligned  |  |
|                  |  |  [AI msg]            left-aligned   |  |
|                  |  |  [User msg]          right-aligned  |  |
|                  |  |  [AI streaming...]   left-aligned   |  |
|                  |  |                                    |  |
|                  |  +------------------------------------+  |
|                  |  +------------------------------------+  |
|                  |  | Input Area (fixed bottom)          |  |
|                  |  | max-width: 768px, centered         |  |
|                  |  | padding: 16px 24px 24px            |  |
|                  |  | [textarea          ] [send btn]    |  |
|                  |  +------------------------------------+  |
|                  |                                          |
+------------------+------------------------------------------+
```

- Sidebar: `width: 240px`, `min-height: 100vh`, `#08090a` bg
- Sidebar/chat divider: `1px solid rgba(255,255,255,0.05)`
- Chat area: `flex: 1`, `#0f1011` bg, `display: flex; flex-direction: column`
- Message list: `flex: 1; overflow-y: auto`, content centered with `max-width: 768px; margin: 0 auto`
- Input area: fixed at bottom of chat area (not viewport), `border-top: 1px solid rgba(255,255,255,0.05)`

### Message Bubble Spec

**User Message (right-aligned):**
- Alignment: `margin-left: auto` (right-aligned within 768px column)
- Max width: 85% of column
- Background: `#191a1b`
- Border: `1px solid rgba(255,255,255,0.08)`
- Border radius: 12px (top-left, top-right, bottom-left), 4px (bottom-right)
- Padding: `8px 16px`
- Text: `#f7f8f8`, 15px weight 400

**AI Message (left-aligned, no bubble):**
- Alignment: `margin-right: auto` (left-aligned within 768px column)
- Max width: 100% of column (text flows naturally)
- Background: none (transparent, sits on `#0f1011`)
- Border: none
- Padding: `4px 0`
- Text: `#d0d6e0`, 15px weight 400
- AI logo at bottom: 16px lucide icon or small SVG, `#62666d` color, `margin-top: 8px`

**Message gap:** 16px vertical between messages

### Input Area Spec

- Container: `padding: 16px 24px 24px`, `max-width: 768px`, centered
- Textarea: `background: rgba(255,255,255,0.02)`, `border: 1px solid rgba(255,255,255,0.08)`, `border-radius: 8px`, `padding: 12px 48px 12px 14px` (right padding for send button)
- Textarea text: `#f7f8f8`, 15px weight 400, placeholder `#8a8f98`
- Textarea min-height: 44px (single line), max-height: 200px (auto-grow)
- Send button: positioned absolute inside textarea right side, `padding: 8px`
  - Enabled: `#5e6ad2` bg circle, `#ffffff` icon, 6px radius (or 50% for circle)
  - Disabled: transparent bg, `#62666d` icon
  - Hover: `#828fff` bg
- Focus: textarea border changes to `rgba(255,255,255,0.12)`

## Data Model Reference

For executor reference — the visual contract depends on this data shape:

```typescript
type Segment =
  | { type: 'text'; content: string }
  | { type: 'tool'; name: string; status: 'calling' | 'done' };

interface Message {
  id: string;
  role: 'user' | 'assistant';
  segments: Segment[];
  timestamp: number;
}
```

Phase 08 only renders `type: 'text'` segments. `type: 'tool'` rendering deferred to Phase 09.

**Source:** ARCHITECTURE.md (Pattern 2: Segments-Based Message Model), CONTEXT D-05
